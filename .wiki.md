# Resumo do Projeto

O projeto **Doce Prazer** é um cardápio interativo online desenvolvido
para uma confeitaria artesanal premium. Ele permite que os clientes
explorem diferentes opções de ovos de chocolate, personalizem seus
pedidos e realizem o pagamento por meio de um QR Code, garantindo uma
experiência de compra simples e intuitiva.

Além disso, os clientes também podem entrar em contato com o suporte e
anexar o comprovante de pagamento durante o processo de pedido.

------------------------------------------------------------------------

# Descrição dos Módulos do Projeto

O projeto inclui diversos módulos funcionais:

-   **Header (Cabeçalho)**: Exibe o logotipo, links de navegação e um
    novo botão de Ajuda.
-   **Hero Section**: Apresenta um banner promocional.
-   **Menu Section (Cardápio)**: Lista os produtos organizados por
    categoria, incluindo opções para ajuste de quantidade e seleção de
    sabores.
-   **Floating Cart (Carrinho Flutuante)**: Mostra um resumo dos itens
    selecionados e o preço total.
-   **Order Modal (Modal de Pedido)**: Exibe um resumo do pedido e o
    valor total, com um botão para prosseguir para o pagamento.
-   **PIX Payment Modal (Modal de Pagamento PIX)**: Gera um QR Code para
    pagamento, mostra a chave PIX e inclui instruções para anexar o
    comprovante.
-   **Integração com WhatsApp**: Permite que os clientes enviem o resumo
    do pedido pelo WhatsApp.
-   **Help Section (Seção de Ajuda)**: Fornece instruções detalhadas
    sobre como realizar um pedido.

# Inventário de Descrição dos Arquivos

-   **index.html** Estrutura principal do cardápio interativo. Inclui
    também o modal de ajuda.

-   **package.json** Lista as dependências do projeto e scripts
    disponíveis.

-   **public/assets/** Contém as imagens utilizadas no projeto.

-   **script.js** Responsável pela lógica do carrinho, processamento de
    pedidos e integração com WhatsApp.

-   **style.css** Contém os estilos de layout e design do cardápio,
    incluindo o modal de ajuda.

-   **template_config.json** Arquivo de configuração para templates.

-   **todo.md** Lista diretrizes de design e tarefas de desenvolvimento.

# Uso do Projeto

Para começar a utilizar o projeto:

1.  **Instalar Dependências** Execute o comando apropriado para instalar
    as dependências listadas no `package.json`.

2.  **Build do Projeto** Utilize uma ferramenta de build, caso
    necessário, para preparar o projeto para implantação.

3.  **Executar a Aplicação** Inicie a aplicação utilizando um servidor
    local.
